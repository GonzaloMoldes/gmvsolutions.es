## 1) Intencion y foco

- [ ] La URL tiene una unica intencion principal.
- [ ] La keyword primaria esta definida y es coherente con el H1.
- [ ] La keyword secundaria apoya sin duplicar intencion de otra URL hermana.
- [ ] La CTA principal es unica y visible en primer scroll.

## 2) Copy y estructura

- [ ] Hero con promesa clara para pyme industrial espanola.
- [ ] Maximo 5 H2 en paginas comerciales.
- [ ] Cada seccion comercial tiene 60-90 palabras maximo.
- [ ] Cada seccion incluye una prueba (dato, ejemplo, captura o proceso).
- [ ] Lenguaje sin keyword stuffing y sin claims no verificables.

## 3) Metadatos

- [ ] Title entre 50 y 60 caracteres.
- [ ] Meta description entre 140 y 155 caracteres.
- [ ] H1 unico y literal con keyword primaria.
- [ ] Canonical definido y revisado.

## 4) Regla canonical blog vs recursos

- [ ] Si el contenido es version completa/editorial: canonical a /blog/*.
- [ ] Si el contenido es landing resumen en /recursos/*: canonical hacia URL fuente definida.
- [ ] La URL canonica coincide con estrategia del cluster (fuente principal + resumen).

## 5) Enlazado interno por cluster

- [ ] La pagina enlaza a su hub principal.
- [ ] La pagina enlaza al menos a 2 URLs hijas relacionadas.
- [ ] La pagina recibe al menos 2 enlaces internos desde el cluster.

## 6) SEO tecnico

- [ ] URL incluida en sitemap.
- [ ] Schema valido para tipo de pagina (SoftwareApplication/Product, Article, FAQPage, BreadcrumbList).
- [ ] Robots/index segun estrategia legal/editorial.
- [ ] Breadcrumbs visibles y coherentes.

## 7) Medicion

- [ ] Evento de CTA principal medido en analytics.
- [ ] URL etiquetada en dashboard semanal.
- [ ] Fecha de publicacion y owner registrados.
