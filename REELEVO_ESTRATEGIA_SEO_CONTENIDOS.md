## 3. Auditoría SEO de la web actual

### Lo que está bien

- **Estructura de URLs limpia**: /recursos/slug-descriptivo/
- **Categorías temáticas**: Gestión del conocimiento, Procedimientos operativos, Gestión del absentismo, Transferencia de conocimiento
- **Casos de uso como páginas**: /casos-de-uso/cobertura-bajas/, onboarding, ETT, transferencia
- **Perfiles de buyer**: /para-quien/jefe-de-produccion/, RRHH, gerente
- **CTA consistentes**: diagnóstico + demo en cada página
- **4 artículos publicados** con buena longitud (10-14 min)

### Lo que hay que mejorar

**En la web existente:**

- **Title tags**: verificar que cada página tiene un title único con la keyword principal. Ejemplo: la home debería ser "REELEVO — Software de documentación operativa para pymes industriales" no solo "REELEVO"
- **Meta descriptions**: cada página necesita una description única de 150-160 chars con keyword + propuesta de valor
- **H1 únicos**: verificar que cada página tiene exactamente un H1 con la keyword objetivo
- **Internal linking**: los artículos del blog deben enlazarse entre sí y a las páginas de casos de uso y /como-funciona/. Ahora mismo cada artículo es una isla
- **Schema markup**: añadir Article schema a los posts del blog, FAQ schema a las páginas de producto, Organization schema a la home
- **Alt text en imágenes**: verificar que las capturas de la app tienen alt text descriptivo ("panel de estadísticas de REELEVO mostrando ejecuciones por proceso")
- **Velocidad**: verificar Core Web Vitals (siendo Next.js + Vercel debería estar bien)

**Palabras clave que repetir más en la web:**

Estas son las keywords que deberían aparecer de forma natural (no forzada) en más páginas:

- "documentación operativa" — en home, cómo funciona, recursos, footer
- "procedimiento operativo" / "SOP industrial" — en home, artículos, casos de uso
- "conocimiento tácito" / "conocimiento del operario" — en casos de uso, artículos
- "pyme industrial" / "taller" / "fábrica" — en toda la web (ahora usa "planta" casi siempre, alternar)
- "QR" / "sin app" / "sin wifi" — diferenciadores clave, repetir en landing y artículos
- "operario sustituto" / "relevo" / "baja" — conecta con el nombre del producto

## 5. Estrategia de internal linking

Cada artículo nuevo debe enlazar a mínimo 3 páginas internas:

```
Artículo SOP → /como-funciona/ + /casos-de-uso/cobertura-bajas/ + otro artículo
Artículo conocimiento → /casos-de-uso/transferencia-conocimiento/ + /para-quien/gerente/ + otro artículo
Artículo formación → /casos-de-uso/onboarding-operarios/ + /precios/ + otro artículo
```

Las páginas de casos de uso y perfiles de buyer deben enlazar de vuelta a los artículos relevantes:

```
/casos-de-uso/cobertura-bajas/ → enlace a artículo plan contingencia + artículo absentismo
/para-quien/jefe-de-produccion/ → enlace a artículo SOP industrial + artículo monitoreo
```

## 6. Optimización on-page — checklist por artículo

Para cada artículo:

- [ ] Title tag: keyword principal + marca (max 60 chars). Ej: "SOP de mantenimiento preventivo: guía + plantilla | REELEVO"
- [ ] Meta description: 150-160 chars con keyword + beneficio + CTA implícito
- [ ] URL: /recursos/slug-con-keyword-principal/
- [ ] H1: contiene keyword principal, único en la página
- [ ] H2s: contienen variaciones de la keyword y keywords secundarias
- [ ] Primer párrafo: keyword en las primeras 100 palabras
- [ ] Imágenes: alt text descriptivo con keyword cuando sea natural
- [ ] Internal links: mínimo 3 a otras páginas del sitio
- [ ] CTA: diagnóstico o demo al final, y uno inline a mitad del artículo
- [ ] Longitud: mínimo 1.500 palabras (los de Whale tienen 2.000-4.000)
- [ ] Schema: Article markup con author, datePublished, dateModified
- [ ] Breadcrumb: Recursos → Categoría → Título

## Resumen ejecutivo

**Whale domina el SEO de "SOP software" en inglés.** Tiene 50+ artículos, reviews en G2/Capterra/Gartner, y una marca reconocida. No puedes competir en inglés ni en keywords genéricas.

**Tu territorio es el español industrial.** Whale no tiene ni una página en español. Nadie está posicionado para "SOP mecanizado", "documentar procesos taller", "procedimiento operativo fabricación" en español. Ese hueco es tuyo.

**La estrategia es sencilla: 4 artículos/mes durante 3 meses, cada uno atacando una keyword de cola larga en español, con contenido específico de planta industrial (no genérico de oficina), con ejemplos reales de mecanizado/soldadura/plegado, y con CTA a diagnóstico o demo.** Al cabo de 3 meses tendrás 16 artículos (los 4 actuales + 12 nuevos) que cubren todo el embudo: desde "qué es un SOP" hasta "comparativa de software" hasta "plantilla descargable".

El SEO tarda 3-6 meses en dar resultados. Pero como la competencia en español para estas keywords es muy baja, puedes estar en primera página mucho antes que si intentaras competir en inglés contra Whale.
